# Kubernetes 용어 사전 (초보자용)

> Kubernetes 프로젝트를 이해하기 위해 알아야 하는 핵심 용어들을 정리한 문서임.
> 각 용어에 실제 소스 코드 위치를 함께 기재하여 코드와 연결할 수 있게 함.

---

## A. Kubernetes 핵심 용어

| 용어 | 설명 | 관련 소스 위치 |
|------|------|---------------|
| **Pod (파드)** | Kubernetes에서 배포 가능한 가장 작은 단위임. 하나 이상의 컨테이너를 포함함. "컨테이너의 묶음"이라고 생각하면 됨 | `staging/src/k8s.io/api/core/v1/types.go` (타입 정의), `pkg/kubelet/pod/` (생명주기 관리) |
| **Node (노드)** | 파드가 실행되는 물리/가상 서버 머신임. 워커 노드와 마스터 노드로 구분됨 | `pkg/controller/nodelifecycle/` (상태 관리) |
| **Cluster (클러스터)** | 여러 노드를 묶어서 하나의 시스템처럼 운영하는 것임 | `hack/local-up-cluster.sh` (로컬 클러스터 구동) |
| **Namespace (네임스페이스)** | 클러스터를 논리적으로 분리하는 가상 구획임. 팀별/환경별로 리소스를 격리할 때 사용함 | `pkg/controller/namespace/` |
| **Deployment (디플로이먼트)** | 파드의 배포와 업데이트를 관리하는 리소스임. "파드를 몇 개 유지해라"를 선언함 | `pkg/controller/deployment/` |
| **Service (서비스)** | 파드에 접근하기 위한 안정적인 네트워크 주소를 제공하는 리소스임. 파드 IP는 변하지만 서비스 IP는 고정됨 | `pkg/controller/service/`, `pkg/proxy/` |
| **ConfigMap (컨피그맵)** | 애플리케이션 설정을 키-값 쌍으로 저장하는 리소스임 | `staging/src/k8s.io/api/core/v1/types.go` |
| **Secret (시크릿)** | 비밀번호, 토큰 같은 민감한 정보를 저장하는 리소스임. 암호화되어 저장됨 | `staging/src/k8s.io/apiserver/pkg/storage/` (암호화 저장) |
| **Volume (볼륨)** | 파드에 연결되는 저장 공간임. 파드가 재시작되어도 데이터가 유지됨 | `pkg/volume/` (20개 서브패키지) |
| **ReplicaSet (레플리카셋)** | 지정된 수의 파드 복제본을 유지하는 리소스임. Deployment가 내부적으로 사용함 | `pkg/controller/replicaset/` |
| **DaemonSet (데몬셋)** | 모든 노드에 파드를 하나씩 배포하는 리소스임. 로그 수집기, 모니터링 에이전트 등에 사용됨 | `pkg/controller/daemonset/` |
| **StatefulSet (스테이트풀셋)** | 상태를 가진 애플리케이션(DB 등)을 순서있게 배포하는 리소스임 | `pkg/controller/statefulset/` |
| **Job (잡)** | 한 번 실행하고 완료되는 작업을 정의하는 리소스임 | `pkg/controller/job/` |
| **CronJob (크론잡)** | 주기적으로 반복 실행되는 작업을 정의하는 리소스임 | `pkg/controller/cronjob/` |
| **Ingress (인그레스)** | 클러스터 외부에서 내부 서비스에 접근하기 위한 HTTP 라우팅 규칙을 정의하는 리소스임 | `staging/src/k8s.io/api/networking/v1/types.go` |

## B. 컴포넌트 관련 용어

| 용어 | 설명 |
|------|------|
| **Control Plane (컨트롤 플레인)** | 클러스터의 "두뇌" 역할을 하는 컴포넌트 집합임. API 서버, 컨트롤러, 스케줄러가 포함됨 |
| **kube-apiserver** | 모든 API 요청을 처리하는 중앙 서버임. Kubernetes의 "대문" 역할을 함 |
| **etcd** | 클러스터의 모든 상태를 저장하는 분산 키-값 데이터베이스임 |
| **kube-controller-manager** | 38개의 컨트롤러를 실행하여 클러스터 상태를 관리하는 컴포넌트임 |
| **kube-scheduler** | 새로 생성된 파드를 적절한 노드에 배치하는 컴포넌트임 |
| **kubelet** | 각 워커 노드에서 파드의 실행을 관리하는 에이전트임 |
| **kube-proxy** | 각 노드에서 서비스 네트워킹 규칙을 관리하는 네트워크 프록시임 |
| **kubectl** | 사용자가 클러스터와 상호작용하기 위한 CLI(명령줄) 도구임 |
| **kubeadm** | Kubernetes 클러스터를 초기 설치하고 설정하는 도구임 |

## C. 빌드/개발 관련 용어

| 용어 | 설명 |
|------|------|
| **Monorepo (모노레포)** | 여러 프로젝트/모듈을 하나의 저장소에서 관리하는 방식임. Kubernetes가 이 방식을 사용함 |
| **staging** | 외부에 독립 라이브러리로 공개될 모듈들이 개발되는 영역임 |
| **vendor** | Go 의존성 소스 코드를 프로젝트 내부에 복사해두는 디렉토리임 |
| **cross-compile (크로스 컴파일)** | 현재 플랫폼과 다른 플랫폼(OS/아키텍처)용 바이너리를 만드는 것임 |
| **Feature Gate (기능 게이트)** | 새 기능을 켜고 끌 수 있는 플래그 시스템임. Alpha → Beta → GA 단계를 거침 |
| **OWNERS** | 각 디렉토리의 코드 소유자(리뷰어)를 정의하는 파일임 |
| **boilerplate** | 모든 소스 파일에 포함되어야 하는 라이선스 헤더 텍스트임 |

## D. 소프트웨어 공학 용어

| 용어 | 설명 |
|------|------|
| **Reconciliation Loop** | "현재 상태"를 "원하는 상태"로 맞추는 무한 루프 패턴임. Kubernetes 컨트롤러의 핵심 동작 방식임 |
| **Watch** | API 서버의 리소스 변경 사항을 실시간으로 구독하는 메커니즘임 |
| **Informer** | Watch를 효율적으로 사용하기 위한 클라이언트측 캐싱 패턴임 |
| **Admission Control** | API 요청을 수정하거나 거부하는 미들웨어 체인임 |
| **CRI (Container Runtime Interface)** | kubelet과 컨테이너 런타임 간의 표준 통신 인터페이스임 |
| **CSI (Container Storage Interface)** | 스토리지 시스템과의 표준 통신 인터페이스임 |
| **CNI (Container Network Interface)** | 네트워크 플러그인과의 표준 통신 인터페이스임 |
| **DFD (Data Flow Diagram)** | 시스템 내 데이터의 이동 경로를 시각화하는 소프트웨어 공학 기법임 |
| **DSM (Dependency Structure Matrix)** | 모듈 간 의존 관계를 행렬로 표현하는 분석 기법임 |
| **Coupling (결합도)** | 모듈 간의 의존 정도를 나타내는 척도임. 낮을수록 좋은 설계임 |
| **Cohesion (응집도)** | 모듈 내부 요소들의 관련성 정도를 나타내는 척도임. 높을수록 좋은 설계임 |
